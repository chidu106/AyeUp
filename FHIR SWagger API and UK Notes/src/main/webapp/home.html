    About

    Overview: FHIR Implementation Guide for Jorvik FHIR



    This site is designed around a fictious NHS Acute trust called Jorvik FHIR. It's purpose is to demostrate examples of current UK implementations.

    This FHIR Messaging specification is based on FHIR DSTU release 2.

    FHIR Profiles, valuesets and Apache Camel Java examples (Apache Tomcat, Swagger and Apache ServiceMix) shown on this site can be found on AyeUp project on GitHub

    Resource Profiles


    Clinical

    • Condition - Get Condition by Id (/Condition/{id})
    • Condition (Bundle) - Find Condition (/Condition)

    Identification

    • Patient - Get Patient by Id (/Patient/{id})
    • Patient (Bundle) - Find Patient (/Patient)

    Documents

    • DocumentReference - Get DocumentReference by Id (/DocumentReference/{id})
    • DocumentReference (Bundle) - Find DocumentReference (/DocumentReference)
    • Binary - Used within Provide Document Bundle
    • Provide Document (Bundle) - Post document and metadata. See also IHE-MHD (/Bundle)
